# 使用手册（门店经营看板）

面向 **店长、收银员、股东、门店老板、超级管理员** 等登录用户，说明常用功能与权限差异。技术部署见 [`deploy-and-maintenance.md`](deploy-and-maintenance.md)，权限模型细节见 [`store-boss-permissions.md`](store-boss-permissions.md)。

---

## 1. 登录与门店

### 1.1 登录

使用管理员分配的 **用户名与密码** 登录。演示环境默认账号见 [`README.md`](../README.md)（如 `admin` / `123456`）；生产环境由超级管理员创建账号。

### 1.2 退出

- 首页左上角 **退出图标**，或  
- **我的** 页面底部 **退出登录**。

### 1.3 当前门店

若账号绑定 **多家门店**，在首页 **「当前门店」** 下拉框中切换。  
**收支、概览、报表、事务、审批** 等均按 **当前所选门店** 隔离；切换门店后数据会随门店变化。

---

## 2. 角色与能看见什么

系统按 **全局角色** 与 **在本店的成员权限**（是否可录入、是否可看概览/流水等）共同决定菜单。下表为常见情况摘要，具体以登录后界面为准。

| 能力 | 超级管理员 | 门店老板 | 店长 / 收银员 / 股东（由老板授权） |
|------|------------|----------|-------------------------------------|
| 经营概况、本月收支、账户详情 | 视是否具备门店与概览权限 | 通常有 | 收银员首页 **不显示** 经营概况/本月收支，其他角色若开通「看概览」则有 |
| 收入 / 支出列表 | 有门店且有权看流水时 | 全店 | 收银员多为 **本人流水**；股东等依授权 |
| 录入收支 | 本店有录入权时 | 有 | 需老板在「我的团队」中勾选 **可录入** 等 |
| 事务管理 | 有 | 依店内权限 | **股东** 在首页 **不显示** 事务入口；其他角色依授权 |
| 发起「修改收支」申请 | 有录入等相关权限时 | 有 | 有录入权等时通常可发起 |
| 审批修改 / 审批事务 | — | 本店老板 | 超级管理员也可 |
| 修改审批（底部导航） | 有 | 本店老板 | 普通店员 **无** |
| 我的 → 用户 / 小店 / 授权 | 有 | **无** | **无** |
| 我的 → 我的团队 | **无** | **有**（本店） | **无** |

**超级管理员** 侧重 **全平台用户、门店、角色**；若未加入任何门店，部分与门店相关的汇总可能为空，属预期行为。

## 3. 底部导航

登录后主界面底部有四个区域（部分页面进入子页后导航会隐藏）：

| 图标 | 含义 |
|------|------|
| 首页 | 返回首页 |
| 图表 | 进入 **营收报表** |
| 中间按钮 | **记一笔**（有录入权限时弹出收入/支出等） |
| 修改审批 | **仅** 门店老板 / 超级管理员可见；进入 **修改审批** 列表 |
| 我的 | 个人中心与系统入口 |

---

## 4. 首页

- **经营概况**：本月收入、支出、净额；可进入 **查看报表** 或点整块进入 **账户详情**。  
- **本月收支**：进入 **本月流水** 列表（可按月份、排序、搜索）。  
- **收入管理 / 支出管理**：查看明细列表，支持日期筛选；有权限时可 **新增** 或 **申请修改** 历史记录。  
- **事务管理**：通知与审批类事项；老板 / 超管可在本页处理 **收支修改审批** 与 **事务单审批**。

---

## 5. 收入与支出

### 5.1 列表与筛选

在收入或支出页使用顶部 **日期筛选**（今日、本周、本月、自定义等）缩小范围。

### 5.2 新增记录

有录入权限时，通过底部 **记一笔** 或列表页的添加入口，填写标题、金额、说明、日期等提交。数据记入 **当前门店**。

### 5.3 修改已记账记录

已入账记录 **不能直接改**，需发起 **修改申请**（说明原因与拟修改内容）。  
**门店老板** 或 **超级管理员** 在 **事务管理** 或 **修改审批** 中 **同意 / 拒绝**；通过后由系统按规则生效。

---

## 6. 账户详情与营收报表

- **账户详情**：在经营概况卡片进入，可查看与本月相关的结构分析（如收入构成等，随版本迭代）。  
- **营收报表**：含趋势、构成、笔数、Top 项目等，数据来自 **当前门店、当前汇总月份** 的收入流水。

---

## 7. 本月流水（月度收支）

选择 **月份**（`YYYY.MM` 形式），查看该月所有收入与支出流水；支持按金额排序、关键词搜索。用于对账与复盘。

---

## 8. 事务管理

- 查看 **通知、审批** 类事务。  
- 若您是 **老板 / 超级管理员**，同一页面可处理：  
  - **收支修改申请**（同意/拒绝）；  
  - **事务单**（如店内审批流，同意/拒绝并可填写原因）。

---

## 9. 修改审批（独立页）

从底部 **修改审批** 进入，列出当前门店待处理与已处理的 **收支修改申请**，可 **同意** 或 **拒绝**。与事务管理中的审批数据同源，便于专门处理「改账」类事项。

---

## 10. 我的

- **我的团队**（**门店老板**）：在本店 **新建店员账号**（店长、收银员、股东等），并设置是否 **可录入**、是否 **可看经营概览**、是否 **可看全店流水** 等。  
- **用户管理 / 小店管理 / 授权管理**（**超级管理员**）：维护全局用户、新建门店并指定店主、配置角色与权限说明等。  
- **系统设置** 中的部分项（如消息通知、支付设置）为界面预留，是否已接通后端以实际环境为准。

## 11. 常见问题

**问：为什么我看不见经营概况？**  
答：常见原因是当前在本店角色为 **收银员**（产品策略为首页仅保留收支入口），或账号 **未开通查看概览**。可联系 **门店老板** 在 **我的团队** 中调整授权。

**问：为什么我只能看到自己的流水？**  
答：**收银员** 默认仅本人数据；**股东** 可能仅本人或依「全店流水」授权。需调整时由 **老板** 修改成员权限。

**问：切换门店后数据变了？**  
答：正常。数据按 **门店** 隔离。

**问：忘记密码？**  
答：联系 **超级管理员** 在用户管理中重置；店内老板通常不能自行改他人密码（以当前版本功能为准）。
